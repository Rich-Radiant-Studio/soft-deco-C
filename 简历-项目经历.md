# 项目经历与工作经历

## 项目一：微前端装修平台系统

### 项目概述
基于 Vue 3 + Wujie 微前端架构的装修行业 SaaS 平台，支持设计师、供应商、施工方、用户等多角色协作，提供从设计到施工的全流程数字化解决方案。

### 项目时间
2025年12月 - 2026年2月

### 技术栈
- **前端框架**: Vue 3 (Composition API + TypeScript)
- **微前端**: Wujie (无界微前端框架)
- **构建工具**: Vite 5.x
- **状态管理**: Pinia
- **路由管理**: Vue Router 4
- **UI 组件库**: Element Plus / Ant Design Vue / Bootstrap 5
- **国际化**: vue-i18n (支持中英西三语)
- **HTTP 客户端**: Axios
- **样式方案**: Scoped CSS + CSS Variables

### 项目架构

#### 系统组成
- **基座应用 (main-web)**: 统一认证、权限管理、子应用加载与通信
- **设计师端 (designer-web)**: 设计工具、模板管理、项目管理
- **供应商端 (supplier-web)**: 供应商管理、订单处理、库存管理
- **施工端 (contractor-web)**: 施工项目管理、工种分类、进度跟踪
- **用户端 (user-web)**: C端用户界面、个人中心、订单管理

#### 微前端架构特点
- 采用 Wujie 框架实现子应用隔离与通信
- 基于 postMessage 实现跨应用无刷新路由跳转
- 统一的语言状态广播机制 (Wujie Bus + localStorage)
- 独立部署、独立开发、技术栈可选

### 核心功能实现

#### 1. 统一认证与权限系统
- **多角色注册登录**: 支持设计师、供应商、施工方、用户、游客五种角色
- **动态表单系统**: 根据角色自动切换注册表单字段
- **游客访问机制**: 
  - 自动生成临时游客账号，无需注册即可访问
  - 游客账号保存与升级功能
  - Token 过期管理 (7天有效期)
  - 路由记忆与恢复
- **RBAC 权限控制**: 基于角色的路由权限管理
- **状态同步**: Pinia + localStorage 实现跨页面状态持久化

#### 2. 国际化系统优化
- **懒加载架构**: 按需加载语言包，减少初始 bundle 体积 60-80%
- **性能优化**: 
  - 代码分割，每个语言包独立打包 (~50-80KB)
  - 首屏加载时间减少 200-500ms
  - 支持语言包预加载和缓存
- **全局同步**: 基座与子应用通过 Wujie Bus 实时同步语言状态
- **性能监控**: 内置语言包加载时间统计和性能分析

#### 3. 响应式设计系统
- **动态尺寸计算**: 
  - 使用 clamp()、vw、vh 等相对单位
  - 基于 CSS Variables 的尺寸系统
  - 自研 useResponsive Composable 工具
- **断点系统**: 支持 xs/sm/md/lg/xl/xxl 六级断点
- **流式布局**: 所有尺寸基于视口动态计算，完美适配各种分辨率

#### 4. C端用户界面 (user-web)
- **首页设计**:
  - 双层导航栏 (注册/语言切换 + LOGO/菜单/搜索/购物车)
  - 左侧筛选栏 (风格/颜色/材料/档次/推荐)
  - 轮播图系统 (自动播放、悬停交互、收藏点赞)
  - 商品网格布局 (响应式两列/单列切换)
  - 促销横幅系统 (每4个商品插入)
- **商品详情页**: 图片画廊、规格选择、评价系统
- **设计师列表与详情**: 设计师展示、作品集、预约功能
- **购物车与订单**: 购物车管理、订单流程、支付集成
- **个人中心**: 用户信息、订单历史、收藏管理

#### 5. 跨应用通信系统
- **消息协议**: 定义 NAVIGATE、LOGOUT 等标准消息类型
- **安全验证**: 基于 trustedOrigins 白名单的消息来源验证
- **导航工具**: 封装 navigation 工具函数，简化跨应用跳转
- **状态共享**: 通过 Wujie Bus 实现子应用间状态共享

### 技术难点与解决方案

#### 难点 1: 微前端子应用通信
**问题**: 子应用需要触发主应用路由跳转，但不能直接访问主应用 router 实例

**解决方案**:
- 采用 postMessage 实现跨应用消息通信
- 定义标准消息协议 (NAVIGATE、LOGOUT)
- 主应用监听消息并执行路由操作
- 封装 navigation 工具函数简化使用

#### 难点 2: 国际化性能问题
**问题**: 三种语言包同步加载导致初始 bundle 过大，影响首屏加载速度

**解决方案**:
- 实现语言包懒加载架构
- 使用 Vite 代码分割功能
- 添加语言包预加载和缓存机制
- 内置性能监控和统计

#### 难点 3: 响应式设计实现
**问题**: 固定像素值导致不同分辨率下显示效果差异大

**解决方案**:
- 制定"所有尺寸动态计算"规范
- 使用 clamp()、vw、vh 等相对单位
- 开发 useResponsive Composable 工具
- 建立 CSS Variables 尺寸系统

#### 难点 4: 游客账号管理
**问题**: 需要支持游客无注册访问，同时提供账号保存和升级功能

**解决方案**:
- 自动生成临时游客账号
- 实现 Token 过期管理 (7天)
- 页面关闭时提示保存账号
- 支持游客升级为正式用户
